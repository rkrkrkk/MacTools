# MacTools

MacTools 是一款原生 macOS 菜单栏工具集合，聚焦高频、轻量、不打扰的系统能力。当前内置物理清洁模式、磁盘清理、阻止休眠、隐藏刘海、显示器亮度与显示器分辨率控制插件，并提供统一的快捷键、功能管理与应用更新体验。

## Product

### Overview

- 菜单栏常驻，默认不进入 Dock，适合日常后台使用。
- 基于插件化功能面板组织能力，支持按需启用、隐藏与排序。
- 菜单栏面板支持主面板直接操作与分层详情交互，可在主面板中完成亮度这类快速调节，也可在侧边详情面板中处理分辨率这类更细粒度的操作。
- 使用 SwiftUI + AppKit 构建，保持接近 macOS 原生交互与视觉风格。
- 支持 macOS 14.0 及以上版本。

### Built-in Plugins

| 插件 | 说明 | 关键能力 |
| --- | --- | --- |
| 物理清洁模式 | 在全部显示器上覆盖黑屏层，并临时禁用输入事件，适合清洁屏幕、键盘或触控板时避免误触与误操作；它不删除磁盘文件。 | 多屏覆盖、退出快捷键水印、辅助功能权限校验、锁屏或睡眠后自动退出 |
| 磁盘清理 | 扫描并清理 macOS 常见缓存。第一版范围限定为缓存清理、开发者缓存清理和浏览器缓存清理，并沿用 Mole `mo clean` 的白名单、路径安全与敏感数据保护规则。 | 干跑扫描预览、可清理/受保护项目区分、执行前二次安全校验、白名单与敏感路径保护 |
| 阻止休眠 | 通过系统级空闲休眠断言保持 Mac 处于唤醒状态，同时允许显示器按照系统策略正常息屏，适合下载、远程连接或长时间后台任务。 | 常驻保持唤醒、30 分钟/1 小时/2 小时/5 小时自动停止 |
| 隐藏刘海 | 自动识别内建刘海屏，并在桌面层遮挡顶部区域，让桌面观感更完整，同时不改动非刘海显示器。 | 仅作用于内建刘海屏、跨 Space 保持生效、切换或更换壁纸后自动保持、开启关闭即时生效 |
| 显示器亮度 | 在菜单栏中直接列出当前可调节亮度的显示器，并为每台显示器快速调节亮度，适合内建屏、支持 DDC 的外接屏以及需要软件调暗的显示器。 | 按显示器独立滑杆、Apple 原生亮度、DDC/CI 亮度控制、Gamma/Shade 软件回退 |
| 显示器分辨率 | 在菜单栏中查看已连接显示器，并为每台显示器切换可用分辨率，适合外接显示器、多屏办公或临时调整显示精细度。 | 按显示器分组浏览、显示当前分辨率、仅展示当前纵横比下的可用模式、侧边详情面板快速切换 |

### Key Capabilities

- 清洁能力区分：清洁模式是物理防误触工具，磁盘清理是缓存文件清理工具，两者入口和行为独立。
- 磁盘清理：第一版只覆盖缓存、开发者缓存和浏览器缓存；扫描阶段不会删除文件，执行阶段会在删除前重新校验路径安全、白名单命中、敏感数据保护和占用状态。
- 隐藏刘海：基于桌面层遮挡实现，不直接改写当前壁纸内容，因此对静态壁纸、动态壁纸、切换 Space 与更换壁纸场景都能保持一致行为。
- 快捷键管理：支持为动作配置快捷键，必要快捷键可重置但不可删除。
- 功能管理：支持插件显示或隐藏，并支持顺序调整。
- 菜单栏面板交互：支持主面板内直接滑杆调节，也支持导航列表与侧边详情联动，适合同时承载显示器亮度与显示器分辨率这类不同层级的功能。
- 应用更新：关于页内置版本信息与更新检查能力，并对磁盘镜像、系统隔离目录等不可更新场景提供明确提示。
- 原生设置页：集中管理权限状态、功能入口、快捷键与应用信息。

## Development & Release

### Requirements

- Xcode
- `xcodegen`

### Quick Start

1. 运行 `make setup` 初始化本地配置和项目基础环境。
2. 编辑 `LocalConfig.xcconfig`，填写 `DEVELOPMENT_TEAM` 和 `BUNDLE_IDENTIFIER_PREFIX`。
3. 本地开发使用 `make run`，如只需编译校验可执行 `make build`。

### Release

1. 复制 `scripts/release.local.env.sample` 为 `scripts/release.local.env`，至少填写 `DEVELOPER_ID_APPLICATION`。
2. 如需 Apple 公证，首次执行一次 `xcrun notarytool store-credentials` 保存凭证，后续可直接复用。
3. 版本号默认读取 `project.yml` 中的 `MARKETING_VERSION` 和 `CURRENT_PROJECT_VERSION`。
4. 生成本地正式包：

```sh
./scripts/release-local.sh
```

5. 如需同步到 GitHub Release，先完成 `gh auth login`，再执行：

```sh
./scripts/release-local.sh --publish
```

更多参数可通过 `./scripts/release-local.sh --help` 查看。
